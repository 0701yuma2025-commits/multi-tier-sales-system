🚀 Supabase セットアップガイド

本番環境のSupabaseプロジェクトをセットアップして、多段階営業代理店管理システムに接続します。

1Supabaseプロジェクト作成

  1. 1. Supabase にアクセス
  2. 2. 「Start your project」をクリック
  3. 3. GitHubアカウントでサインアップ（推奨）
  4. 4. 「New Project」をクリック
  5. 5. 以下の情報を入力：
    • Project name: multi-tier-sales-system
    • Database Password: 強力なパスワードを生成
    • Region: Northeast Asia (Tokyo)を選択
  6. 6. 「Create new project」をクリック
⏳ プロジェクト作成には約2分かかります。作成が完了したら次のステップに進んでください。

3認証メール設定（重要）

確認メールのリダイレクトURLを設定する必要があります：

  1. Supabaseダッシュボードで「Authentication」→「URL Configuration」へ移動
  2. 「Site URL」に以下を設定： 
    https://your-domain.com/
    ※ローカル開発の場合：http://localhost:3000/
  3. 「Redirect URLs」に以下のURLを追加（複数可）： 
    http://localhost:3000/login.html
http://localhost:3000/reset-password.html
https://your-domain.com/login.html
https://your-domain.com/reset-password.html
  4. 設定を保存
⚠️ この設定をしないと、確認メールのリンクをクリックしても正しいページに戻れません

4データベーステーブル作成

以下のSQLをSupabaseのSQL Editorで実行してください：

-- UUIDエクステンションを有効化
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- 代理店テーブル
CREATE TABLE agencies (
    id UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
    company_name VARCHAR(255) NOT NULL,
    company_type VARCHAR(20) CHECK (company_type IN ('法人', '個人')),
    representative JSONB NOT NULL,
    tier_level INTEGER CHECK (tier_level BETWEEN 1 AND 4),
    parent_agency_id UUID REFERENCES agencies(id),
    status VARCHAR(20) DEFAULT 'pending',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 商品テーブル
CREATE TABLE products (
    id VARCHAR(10) PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    price INTEGER NOT NULL,
    base_rate DECIMAL(5,2),
    tier_rates JSONB,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 売上テーブル
CREATE TABLE sales (
    id VARCHAR(20) PRIMARY KEY,
    agency_id UUID REFERENCES agencies(id),
    product_id VARCHAR(10) REFERENCES products(id),
    customer_name VARCHAR(255) NOT NULL,
    amount INTEGER NOT NULL,
    status VARCHAR(20) DEFAULT 'confirmed',
    sale_date DATE NOT NULL,
    notes TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 報酬テーブル
CREATE TABLE commissions (
    id UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
    sale_id VARCHAR(20) REFERENCES sales(id),
    agency_id UUID REFERENCES agencies(id),
    amount INTEGER NOT NULL,
    rate DECIMAL(5,3),
    type VARCHAR(20),
    status VARCHAR(20) DEFAULT 'unpaid',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

6Row Level Security (RLS) 設定

セキュリティのためRLSを有効化します：

-- 各テーブルでRLSを有効化
ALTER TABLE agencies ENABLE ROW LEVEL SECURITY;
ALTER TABLE products ENABLE ROW LEVEL SECURITY;
ALTER TABLE sales ENABLE ROW LEVEL SECURITY;
ALTER TABLE commissions ENABLE ROW LEVEL SECURITY;

-- ポリシー作成（一旦全て許可 - 後で調整）
CREATE POLICY "agencies_select_policy" ON agencies FOR SELECT USING (true);
CREATE POLICY "products_select_policy" ON products FOR SELECT USING (true);
CREATE POLICY "sales_select_policy" ON sales FOR SELECT USING (true);
CREATE POLICY "commissions_select_policy" ON commissions FOR SELECT USING (true);
